Discrete random variables

Recall the definition of a binomial distribution:

A random variable \(X\) is said to have a binomial distribution with size parameter \(n \in \mathbb{N}\) and probability parameter \(p \in (0,1)\) if its mass function is \[f(x) %= \frac{n!}{x!(n-x)!}\,p^{x}\,(1-p)^{n-x} = \binom{n}{x}\,p^{x}\,(1-p)^{n-x}\] for \(x \in \{ 0,1,2,\ldots n\}\) and \(f(x) = 0\) otherwise. We write \(X\sim \text{Bin}(n, p)\).

Sampling and simulation

We have seen above how to use R to check the formulae for the expectations of various discrete probability distributions. We cant do the same for continuous random variables because the expectation and variance of continuous variables are defined using integrals. For example, the density function of an exponential random variable is \[f(x) = \lambda e^{-\lambda x}\] for all \(x > 0\). We claimed in lectures that its expectation is \[{\mathsf E}(X) = \frac{1}{\lambda}\,.\]

We can instead use the function rexp() to take a random sample from our exponential distribution. If we take the mean of a very large sample, it seems sensible to assume that the sample mean will probably be close to the expectation (the population mean) of the distribution.

lambda <- 2.5

x <- rexp(n = 100, rate = lambda)    # A sample of size n = 100.

mean(x)
## [1] 0.3782029
1/lambda
## [1] 0.4

What do you think happens if we repeat this code? Will we see exactly the same result? What do you think happens if we increase the sample size? Discuss your thoughts.

This is a very simple example of how a simulation study is carried out: rather than deriving properties of distributions mathematically, we can check them by seeing if they approximately hold for large samples drawn from that distribution. This can be especially helpful in situations where a property seems intuitively clear but the maths required to prove the property is extremely difficult. Simulations are also a useful way to check that our maths is correct once it is done.

f = function(x, n, p){
   return( (p^x)*((1-p)^(n-x))*factorial(n)/(factorial(x)*factorial(n-x)))

Solution to Exercise 5

The important difference here is that a Poisson distribution can take infinitely many values, and we cant sum over all of them in R. At best, we can sum over many values and hope to see answers that are approximately correct.

x <- 0:1000  # lots of possible values

sum(dpois(x, lambda = 2.5))
## [1] 1
E <- sum(x*dpois(x, lambda = 2.5))
E
## [1] 2.5
# And we can check that the variance is equal to lambda too:

var <- sum( (x - E)^2 * dpois(x, lambda = 2.5) )
var
## [1] 2.5

Figure 4: Normal density curves for \(X\), \(Y\) and \(X + Y\). We can see that the curve for \(X + Y\) has a greater spread (its variance is the sum of the variances for \(X\) and \(Y\)) and its location is further to the right (its expectation is the sum of the expectations for \(X\) and \(Y\)).

In this graph, we see the normal density curves for \(X\) and \(Y\) to the left. We see the curve for \(X + Y\) to the right with, as expected, a higher mean (\(10 + 20\)) and a higher spread (the variance should be \(5^2 + 10^2\)).
